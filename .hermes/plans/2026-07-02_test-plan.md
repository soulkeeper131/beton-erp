## Част A: Архитектурна документация

### A1. Tech Stack
| Слой | Технология | Защо |
|------|-----------|------|
| Frontend | Next.js 14 App Router + React 18 | SSR/SSG, file-based routing, API routes в същия проект |
| UI | Tailwind CSS + shadcn/ui | Utility-first CSS, готови accessible компоненти |
| Auth | NextAuth.js v5 (Credentials + JWT) | Безопасна сесийна автентикация, JWT токени, Edge-ready |
| DB | SQLite (better-sqlite3) + Drizzle ORM | Zero-config, file-based, типобезопасен ORM с миграции |
| PDF | @react-pdf/renderer | Server-side PDF генериране с React компоненти |
| Email | Nodemailer | SMTP клиент за изпращане с PDF attachment |
| Hosting | Coolify + Docker | Self-hosted PaaS, auto-deploy от GitHub |

### A3. DB Схема (22 таблици)
- **users** — admin | employee роли
- **clients** — име, ЕИК, ДДС, адрес
- **sites** — обекти с град/село, статус
- **offers + offer_items** — оферти с бетон/услуги
- **invoices + invoice_items** — фактури (вх/изх, EUR, ЗДДС)
- **pourings + act_workers + act_materials** — актове за бетониране
- **machines + machine_maintenance** — автопарк + ремонти
- **workers + worker_attendance** — работници + присъствие
- **materials + material_deliveries** — склад + доставки
- **services + service_items** — комплексни услуги
- **site_calendar** — планиране на наряди
- **act_photos** — снимки към актове/обекти
- **audit_log** — одитен журнал
- **company_settings** — singleton фирмени данни + SMTP
- **templates** — PDF шаблони
- **api_keys** — API ключове

### A4. Auth Flow
1. Потребител въвежда email/парола → NextAuth Credentials provider
2. bcryptjs проверява хеша → връща JWT с {id, email, name, role}
3. JWT token в cookie → middleware проверява за защитени рутове
4. UI: useSession() за role-based conditional rendering
5. API: ръчна auth() проверка във всеки route handler
