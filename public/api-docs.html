📡 API Документация

REST API за Beton ERP — пълна справка на всички ендпойнти

🔐 Автентикация

Всички защитени ендпойнти изискват едно от следните:

  • Session cookie (от логнат потребител)
  • Authorization: Bearer <API_KEY> header

# Session-based
curl https://beton.blv.bg/api/clients

# API Key
curl -H "Authorization: Bearer *** /api/clients

🏢 Клиенти

GETauth
/api/clients
Списък на всички клиенти
curl https://beton.blv.bg/api/clients
POSTauth
/api/clients
Създаване на нов клиент. EIK се валидира автоматично през CompanyBook.
curl -X POST https://beton.blv.bg/api/clients \
  -H "Content-Type: application/json" \
  -d '{"name":"Иван","companyName":"Фирма ЕООД","eik":"123456789","email":"info@firma.bg"}'
GETauth
/api/clients/:id
Детайли за конкретен клиент

📦 Каталог

GETauth
/api/catalog
Структуриран каталог — бетони, услуги, машини, материали
curl https://beton.blv.bg/api/catalog
# {"concreteTypes":[...], "services":[...], "machines":[...], "materials":[...]}

🚛 Машини, 👷 Работници, 📦 Склад, 🛠️ Услуги, 🧱 Бетон

Всички тези модули следват еднакъв CRUD pattern:


GET /api/machines — списък
POST /api/machines — създаване
GET /api/machines/:id — детайли
PUT /api/machines/:id — редакция
DELETE /api/machines/:id — изтриване

Същото за: /api/workers, /api/materials, /api/services, /api/concrete-types
